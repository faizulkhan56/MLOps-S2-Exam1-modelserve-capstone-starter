Fresh VM setup for Phase 1–4 recovery (and Phase 5–6 prerequisites)

One-time on a brand-new Poridhi VM (not only git pull):
- Install Docker Engine + Compose plugin (or use course image if preinstalled).
- If `docker` needs sudo: `sudo usermod -aG docker $USER` then log out and back in, or use `sudo docker compose ...` for all compose commands.
- You still need: venv, `pip install -r requirements.txt`, `.env`, Kaggle data, train, Feast apply/materialize, and (for Phase 5+) a rebuilt `api` image after code changes.

Step 1: Clone / pull project
cd ~/code
git clone <YOUR_GITHUB_REPO_URL>
cd MLOps-S2-Exam1-modelserve-capstone-starter

If already cloned:

cd ~/code/MLOps-S2-Exam1-modelserve-capstone-starter
git pull
Step 2: Create Python venv
sudo apt update
sudo apt install -y python3-venv python3-pip python-is-python3 unzip
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
Step 3: Create .env
cp .env.example .env
Step 4: Start Docker stack
docker compose up -d --build
docker ps

Validate:

curl http://localhost:8000/health
curl http://localhost:5000
curl http://localhost:9090/api/v1/targets
curl http://localhost:3001/api/health

Redis (no host redis-cli needed):

docker compose exec redis redis-cli PING
# expect PONG
Kaggle dataset setup
Step 5: Install Kaggle CLI
pip install kaggle
Step 6: Create Kaggle token file
mkdir -p ~/.kaggle
echo '{"username":"YOUR_KAGGLE_USERNAME","key":"YOUR_KAGGLE_KEY"}' > ~/.kaggle/kaggle.json
chmod 700 ~/.kaggle
chmod 600 ~/.kaggle/kaggle.json

Check:

cat ~/.kaggle/kaggle.json
Step 7: Download dataset
mkdir -p data/raw
cd data/raw
kaggle datasets download -d kartik2112/fraud-detection

ls -lh

Return to project root:

cd ~/code/MLOps-S2-Exam1-modelserve-capstone-starter
Phase 3 recovery: train and register model
Step 8: Wait for MLflow, then train
source .venv/bin/activate
python scripts/wait_for_mlflow.py

Optional faster first run (smaller CSV load):

# export TRAIN_MAX_ROWS=50000

python training/train.py

Verify registered model:

python - <<'PY'
from mlflow.tracking import MlflowClient

client = MlflowClient(tracking_uri="http://127.0.0.1:5000")

models = client.search_registered_models()
if not models:
    print("No registered models found")
else:
    for model in models:
        print("Model:", model.name)
        for version in model.latest_versions:
            print("  Version:", version.version, "Stage:", version.current_stage)
PY

Expected:

Model: modelserve_classifier
  Version: <n> Stage: Production   (version increments if you retrain)

Phase 5–6 on this VM (after pull / code changes)

- The API container uses `MLFLOW_TRACKING_URI=http://mlflow:5000` and `FEAST_REPO_PATH=/app/feast_repo` (baked into the image at build time). After you pull new app code or change `feast_repo/`, rebuild and restart the API:
  docker compose build api
  docker compose up -d api
- Run API tests from the repo root (venv active):
  pytest app/tests/ -q
- Smoke the service:
  curl -s http://localhost:8000/health
  curl -s http://localhost:8000/metrics | head
