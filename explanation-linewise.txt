24. Very important conceptual answer
Does training depend on Feast?

In your current code:

No, training does not read from Feast.

Training creates:

features.parquet

Then Feast uses that later.

So:

train.py → creates Feast input
Feast → serves features during inference
Does API depend on Feast?

Yes.

main.py depends on Feast through:

feats = _feast_client.get_features(req.entity_id)

If Feast/Redis does not have that entity ID, prediction fails.

Why no Feast container?

Because Feast is:

Python SDK + CLI

not a long-running service.

The long-running storage backend is:

Redis

So your runtime is:

API container includes Feast library
Redis container stores online features
